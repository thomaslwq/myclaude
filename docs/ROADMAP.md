## 二、迭代计划（5 个里程碑）

### 🟢 P0 — 激活已有能力（1-2 周）✅

**目标**：把隐藏的好东西开放出来，形成差异化竞争力

| 任务 | 说明 | 状态 |
|------|------|------|
| **开放 BUDDY 系统** | 将 `feature('BUDDY')` 默认启用，用户第一次启动时自动获得一只宠物 | ✅ |
| **完善 `/buddy` 命令** | 添加孵化引导、卡片展示、互动反馈 | ✅ |
| **BUDDY AI 上下文** | 宠物性格影响 AI 回复风格（宠物属性→AI temperature/personality） | ✅ |

### 🟡 P1 — 成长系统（2-4 周）✅

**目标**：编辑器和使用者真正"一起成长"

| 任务 | 说明 | 状态 |
|------|------|------|
| **BUDDY 养成深化** | 宠物经验值系统 — 根据用户使用频率和时长升级，进化形态 | ✅ |
| **成就系统** | 解锁成就徽章：连续使用7天、重构100次、第一次发布npm包等 | ✅ |
| **编码统计看板** | 统计用户的编码语言分布、常用命令、效率趋势图 | ✅ `/mystats` |
| **技能推荐引擎** | 根据用户常用操作推荐相关技能 | ⬜ |
| **编码习惯学习** | Auto Memory 从会话中自动提取用户的编码偏好 | ⬜ |

### 🟠 P2 — 互动体验提升（4-6 周）🔄

**目标**：让终端不再冰冷，充满有温度的互动

| 任务 | 说明 | 状态 |
|------|------|------|
| **BUDDY 多交互模式** | 触摸（pet）、喂食（feed）、玩耍（play）、教学（teach） | ✅ feed/play |
| **智能提示卡片** | 在终端空闲时，宠物携带实用小贴士出现 | ✅ |
| **BUDDY 记忆联结** | 宠物记住用户的重要事件（第一次提交、发布版本）并主动提及 | ⬜ |
| **节日/彩蛋系统** | 生日、节日、使用里程碑触发特殊动画 | ⬜ |
| **语音互动** | 利用现有 `src/voice/` 基础设施，支持语音命令（实验性） | ⬜ |

### 🔵 P3 — 扩展生态（待启动）

| 任务 | 说明 | 状态 |
|------|------|------|
| **插件市场** | 搭建插件市场网站，用户可以发布和安装社区插件 | ⬜ |
| **Skill Studio** | 图形化技能编辑器，让用户能自己编写技能（拖拽式） | ⬜ |
| **BUDDY 插件化** | 宠物本身成为插件系统的一部分，第三方可以开发新物种、新装备 | ⬜ |
| **MCP 生态集成** | 预置常用的 MCP 服务器：GitHub、Jira、Docker、数据库 | ⬜ |
| **自定义主题市场** | 用户分享终端主题和配色方案 | ⬜ |

### 🟣 P4 — 智能进化（待启动）

| 任务 | 说明 | 状态 |
|------|------|------|
| **个人模型微调** | 基于用户的编码历史，生成个性化的 system prompt | ⬜ |
| **智能上下文管理** | 自动学习哪些项目文件最常访问，优化上下文窗口分配 | ⬜ |
| **协作模式** | 多用户共享 BUDDY、共享记忆（团队级记忆） | ⬜ |
| **离线模式** | 本地小模型（通过 Ollama）支持基本命令，不依赖 API | ⬜ |
| **BUDDY 遗传进化** | 宠物基因系统，不同宠物可以培育出新品种 | ⬜ |

## 三、优先级 vs 投入矩阵

```
高 │
影 │ P0 ───── P1 ───── P2
响 │  开放BUDDY   成长系统   互动体验
力 │
   │ P3 ───────── P4
   │  扩展生态      智能进化
低 │
   └─────────────────────────
     低    实施难度     高
```

**建议执行顺序**：
1. **P0（立刻）** — 低投入高回报，激活已有功能
2. **P1（近期）** — 核心差异化，建立壁垒
3. **P2 + P3 并行（中期）** — 体验 + 生态
4. **P4（长期）** — 基础设施投入

## 四、关键差异化优势

| vs 原版 Claude Code | myclaude 差异化 |
|---------------------|----------------|
| 纯工具 | **有温度的 AI 伙伴** — BUDDY 让你愿意打开终端 |
| 无记忆 | **跨会话学习** — 越用越懂你 |
| 封闭生态 | **开源 + 插件市场** — 社区驱动 |
| 单一模型 | **多模型 + MCP** — 灵活替换 |
| 用完即走 | **持久模式 KAIROS** — 后台运行，主动帮你 |
